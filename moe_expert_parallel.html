MoE: Expert Parallelism + FSDP

2 ranks, each owning one expert. Non-MoE layers (attention) use normal FSDP: shards are AllGathered into a transient model buffer for compute, producing saved activations — just like the e2e demo. Expert layers skip AllGather — each rank holds its expert's full weights. All-to-All moves tokens to the owning rank for compute.

Note: The FSDP paper (VLDB 2023) does not describe MoE or expert parallelism. For sparse parameters (e.g. DHEN embeddings), the paper uses an ignore-list so FSDP only handles dense layers, while a separate system shards the sparse tables. This visualization depicts dp2ep expert parallelism from PyTorch's torchtitan (2024–25), which composes with FSDP on a 2D device mesh. With more ranks, each expert column would itself use intra-column FSDP (AllGather → GEMM → ReduceScatter) — simplified here to 1 rank per column.
